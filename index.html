The problem

Human brains are bad at
exhaustive enumeration

You can read two rules separately and both sound reasonable. The contradiction only appears when you force both conditions to hold simultaneously — which almost nobody does across 10+ rules written by different people over months.

"I have been using Linux since a little longer than 6 years now, but I have been using Mac since close to 3 years. If I did not use Linux, I would have never switched to Mac. But me using Mac has no relation with me using Linux in any way."

Sentence 3 says Linux caused the switch to Mac. Sentence 4 says they are completely unrelated. Both cannot be true. The engine finds this in milliseconds.

python
# L = using Linux · M = using Mac
# Claim 3: !L → !M  (equivalent to: Mac requires Linux)
# Claim 4: M and L are independent

result = check_prompt_logic(["L+!M", "M"])
# → conflicting_pairs: 1
# → L=0, M=1 satisfies "M" but violates "L+!M"
# → both cannot be simultaneously true

LLMs have the same failure mode. They predict what sounds right. Simple contradictions they catch. Five nested conditions with a double negation — they'll confidently give the wrong answer. The engine doesn't read. It computes every combination. The contradiction is unavoidable.
What it does

Expression in. Exact answer out.

Forward evaluation, inverse synthesis, and rule auditing. All deterministic. All mathematically exact.

Forward · Expression → Truth Table

python
from core.evaluator import evaluate

Inverse · Truth Table → Minimal Expression

python · Quine-McCluskey
from core.synthesizer import synthesize

# Prove distributive law
t1, _ = evaluate("A.(B+C)")
t2, _ = evaluate("A.B+A.C")

t1.minterms == t2.minterms  # True

minimal, _ = synthesize(t1)
# → "A.C+A.B"

Credibility

Not a claim. A consequence
of how it works.

Exhaustive enumeration. For n variables it evaluates exactly 2ⁿ rows. Nothing is sampled or predicted.

ResultWhat it means
SatisfiableFound an actual row where output = 1
ContradictionChecked every row — all were 0
TautologyAll rows output 1 — rule is redundant
EquivalentOutput columns identical row-by-row
ConflictConjunction always false — can never both fire

The core evaluator is 15 lines of code. If the operators are correct, the results are correct. No black box. Fully auditable.

scale
 5 variables →       32 rows  CPU
10 variables →    1,024 rows  CPU
15 variables →   32,768 rows  CPU
20 variables → 1,048,576 rows  numpy
50+ variables →    10^15 rows  CUDA

# Each row is completely independent.
# No row needs the result of any other.
# 1 GPU thread = 1 row.

For the target use case — system prompt rules, business logic, access control — 5–15 variables is the norm. Provably complete. Stronger than any probabilistic tool.

Benchmark

Every model hallucinates on boolean logic.

We tested 5 models. All of them got logic questions wrong. The pattern tells you why you need a deterministic layer.

Model Size Hallucination rate Pattern
tinyllama 1.1B 50% Always answers "yes" — not reasoning, picking a default
llama3.2:3b 3B 50% Always answers "no" — not reasoning, picking a default
gemma3:4b 4B 35% Reasons per case, wrong 1 in 3
qwen3-32b 32B 17% Consistent ~17% baseline across all complexity levels

The small models aren't reasoning at all. The large models reason but still hallucinate at a consistent rate. Complexity doesn't make it worse — qwen3-32b was flat at 16–19% from 3 variables (8 rows) to 10 variables (1,024 rows). The errors are structural, not difficulty-dependent.
